# Конфигурация "Управление IT-фирмой". Кастомизация для компании "Ваш компьютерный мастер". Версия 0.9 
 
Данная конфигурация создана для компании "Ваш компьютерный мастер" и имеет дополнительный функционал:

1) Подсистема "Работа с заявками клиентов", с реализацией уведомления специалистов обслуживания с использованием Telegram-бота + отчет "Анализ выставленных актов"
2) Подсистема "Расчет управленческой зарплаты" + отчеты "Выработка специалистов", "Расчёты с сотрудниками", "Расход запланированных отпусков"

Для работы в конфигурации необходимо провести начальную настройку программы пользователю с правами Администрирования. 

### Порядок начальной настройки программы:

<details>
<summary><b>1. Создайте Telegram-бот и получите токен бота</b></summary>

  
  1.1 Напишите в Telegram `https://t.me/BotFather` команду "/start"

  1.2 Напишите команду "/newbot"

  1.3 Выберите имя вашего бота, которое будут видеть пользователи

  1.4 Выберите идентификатор вашего бота (он должен заканчиваться на "bot")

  1.5 Скопируйте (сохраните) токен, который пришлет BotFather
   
</details> 

<details>
<summary><b>2. Создайте группу в Telegram и получите идентификатор группы</b></summary>

 
  2.1 Создайте группу в Telegram

  2.2 Добавьте в группу только что созданного бота

  2.3 Назначьте боту права администратора

  2.4 Напишите любое сообщение в группу

  2.5 С помощью браузера или Postman выполните запрос `https://api.telegram.org/bot[ВашТокен]/getUpdates`. В полученном JSON найдите идентификатор группы, в которой получено сообщение, и сохраните его в константу.
   
</details> 

 <b>3. Заполните значения констант:</b>
 
 - *"Номенклатура Абонентская плата"* и *"Номенклатура Работы специалиста"* – подходящие по смыслу наименования услуг из справочника "Номенклатура". В случае отстутсвия создать необходимую номенклатуру
 - *"Токен управления телеграм ботом"* – токен Telegram-бота
 - *"Идентификатор"* группы" – идентификатор группы Telegram
 - *"НДФЛ"* - величина текущего процента НДФЛ для расчета удержаний

<details>
<summary><b>4. Заполните данные графика работы</b></summary>

  
  4.1 В справочнике "Графики работы" добавить график работы (например, "Пятидневка")

  4.2 В подменю Сервис подсистемы "Добавленные объекты" открыть обработку "Запонение графика". Заполнить поля
     
     - *"Период"* - Укажите период заполнения графика работы
     - *"График работы"* - Укажите необходимый график из предложенного списка
     - *"Выходные дни"*  - Укажите выходные дни в формате чисел, соотвествующих порядковому номеру дня недели, через запятую (например - "6,7" для субботы и воскресенья)

  4.3 Нажмите кнопку **Заполнить**

</details>

*Рекомендации: для расчета отпусков внести отдельный график без выходных дней с наименованием "Отпуск"*

**5. Создайте пользователей для каждого сотрудника, использующего 1С**.  Назначьте права доступа в соотвествии с должностными обязанностями сотрудников в разделе *"Администрирование" – "Настройки пользователей и прав" – "Профили групп доступа"*

**6. Внесите данные о сотрудниках.** Заполните регистр сведений *"Условия оплаты сотрудников"*. Для категории *"Специалист"* укажите процент, начисляемый за выполнение работ. Для иных категорий поле процента работ не имеет пользовательской видимости. Должности указываются при заполнении справочника *Сотрудники*.


**7. Проверьте статус регламентного задания *"Отправка уведомлений в Телеграм"* (должно быть **включено**).** Установите настройки расписания уведомлений.

**8. Заполните дополнительные поля *"Период действия договора"*, *"Сумма абоненсткой платы"* и *"Стоимость часа"* у вновь введенного вида договоров *"Абонентское обслуживание"*.**
